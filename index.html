Memory Systems

Five layers, one unified interface

Each memory type solves a different persistence problem. Together they give agents continuous context across sessions, machines, and collaborators.

🏠

Hermes Local Memory

File-based memory stored on the agent's machine. Tracks user preferences, behavioral corrections, project-specific conventions, and agent configuration. Private and instant — no network call needed.

LOCAL · PRIVATE
🌐

Open Brain Vector Memory

Supabase-hosted pgvector store shared across every connected agent. Stores evergreen facts, project decisions, and domain knowledge. Semantic search finds the right memory without exact keyword match.

SHARED · VECTOR SEARCH
📓

Session Search

Full-text search over past conversation transcripts. Lets agents recall what was discussed, decided, or debugged in previous sessions — even months later. Context continuity across conversation boundaries.

TRANSCRIPTS · RECALL
✅

Google Tasks

Actionable to-do items surfaced during conversations. Agents can create, update, and complete tasks that sync with Google Tasks — turning conversational intent into tracked work items.

ACTIONS · TRACKED
⚙️

Hermes Skills

Reusable, parameterized procedures the agent can invoke by name. From "review a PR" to "deploy staging" — skills encode repeatable workflows that get smarter over time.

PROCEDURES · REUSABLE
